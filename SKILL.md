## 技能概述

本技能提供自动化从中国知网(CNKI)批量下载学术论文的能力。通过浏览器自动化和智能解析，支持11种文献类型的高效获取，具备完善的错误处理和稳定可靠的下载流程。

## 6大绝对禁止原则

针对 CNKI 论文下载的定制化禁止原则：

```yaml
1. 禁止绕过付费限制
   说明: 不尝试破解或绕过CNKI的付费机制，自动跳过需要付费的论文
   实现: 检测付费提示，记录跳过原因，不使用任何破解手段

2. 禁止过度频繁请求
   说明: 不进行可能导致IP被封禁的高频请求，合理控制并发数量
   实现: 串行下载，每篇之间延迟5秒，默认并发数不超过3

3. 禁止忽视版权限制
   说明: 不下载超出用户权限范围的论文，尊重知识产权
   实现: 遵守机构IP限制和账号权限，不尝试突破权限边界

4. 禁止破坏网站结构
   说明: 不使用可能对CNKI网站造成影响的恶意爬取方式
   实现: 使用标准Playwright API，模拟真实用户行为，不注入恶意代码

5. 禁止伪造用户身份
   说明: 不模拟或伪造机构账号，使用真实的用户登录状态
   实现: 使用用户自己的浏览器会话，不伪造登录凭证

6. 禁止忽略数据完整性
   说明: 不接受部分下载或损坏的文件，确保每篇论文完整下载
   实现: 验证文件大小（>100KB），检查文件可读性，失败自动重试
```

## 任务分解规则

### 三层分解法

```yaml
第一层: 主要阶段
  - 阶段1: 需求解析
  - 阶段2: 浏览器操作
  - 阶段3: 串行下载
  - 阶段4: 结果报告

第二层: 每阶段的子任务
  - 任务1.1: 解析自然语言输入
  - 任务1.2: 验证参数完整性
  - 任务1.3: 确定文献类型
  - 任务2.1: 打开CNKI首页
  - 任务2.2: 选择文献类型
  - 任务2.3: 执行检索
  - 任务2.4: 获取论文列表
  - 任务3.1: 串行下载控制
  - 任务3.2: 文件保存
  - 任务3.3: 错误处理
  - 任务4.1: 生成统计报告
  - 任务4.2: 记录下载日志

第三层: 可执行原子任务
  - 打开浏览器
  - 定位搜索框
  - 输入关键词
  - 点击搜索按钮
  - 等待结果加载（8-10秒）
  - 提取论文信息
  - 点击下载按钮
  - 保存PDF文件
  - 更新进度
  - 记录日志
```

### 任务创建规则

```yaml
规则1: 每个下载任务必须独立可执行
规则2: 任务之间明确依赖关系（先检索后下载）
规则3: 每个任务有明确的验收标准（文件完整、无损坏）
规则4: 任务粒度适中（单篇下载0.5-2分钟）
规则5: 自动化任务优先（减少用户干预）
```

### PDF下载策略

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `pdf_preferred` | bool | true | 是否优先下载PDF格式 |
| `pdf_only` | bool | false | 是否仅下载PDF（false时PDF不可用降级到CAJ） |
| `skip_paid_papers` | bool | true | 是否跳过需要付费的论文 |

## 核心优化经验

### 等待策略优化

```yaml
问题: CNKI页面加载慢，networkidle经常超时
解决:
  1. 使用 domcontentloaded 代替 networkidle
  2. 添加固定等待（8-10秒）
  3. 增加超时时间到 45000ms

效果:
  成功率: 60% → 95%+
  超时频率: 每3-5篇1次 → 几乎不超时
```

### 下载流程优化

```yaml
问题: 并发下载导致 AbortError
解决:
  1. 改为串行处理
  2. 每篇论文之间延迟5秒
  3. 每步操作后验证状态

效果:
  稳定性: 经常失败 → 稳定完成
  用户体验: 卡住频繁 → 进度清晰
```
